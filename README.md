### 🧠 Backend Developer로서 중요하게 생각하는 것

**사용자가 안정적으로 사용할 수 있는 서버를 만드는 것을 중요하게 생각합니다**

백엔드는 화면에 직접 보이지는 않지만, 서비스가 빠르고 안정적으로 동작하기 위해 꼭 필요한 부분이라고 생각합니다.  
그래서 단순히 기능을 구현하는 것에서 끝내지 않고, 오래 실행되어도 문제가 없고 다른 사람이 이해하기 쉬운 코드를 작성하려고 노력합니다.

메모리 관리를 잘한다는 것은 단순히 메모리를 적게 쓰는 것이 아니라, 서버가 오래 실행되어도 안정적으로 동작하도록 객체 생성, 자원 반납, 대량 데이터 처리, 캐시 사용, JVM 동작 방식을 함께 고려하는 것이라고 생각합니다.

- ⚙️ **안정적인 서버 운영** — 서버는 계속 실행되기 때문에 작은 오류나 메모리 누수도 장애로 이어질 수 있다고 생각합니다. JVM 메모리 구조, Garbage Collection, Thread 관리에도 관심을 가지고 있습니다.
- 🚀 **효율적인 자원 사용** — 메모리, DB Connection, File I/O 같은 자원을 낭비하지 않고 필요한 만큼만 사용하려고 합니다. Connection Pool과 `try-with-resources`처럼 자원을 안전하게 관리하는 방식도 중요하게 생각합니다.
- 📄 **대량 데이터 처리 고려** — 많은 데이터를 한 번에 가져오기보다 Pagination, Batch Processing을 활용하고, Lazy Loading은 N+1 문제 같은 부작용까지 고려해 상황에 맞게 사용하려고 합니다.
- 🧹 **읽기 쉬운 코드 작성** — 나중에 다시 보거나 다른 사람이 봐도 흐름을 이해할 수 있도록 Layered Architecture, DTO, Service, Repository처럼 역할이 분리된 구조를 추구합니다.
- 🧱 **객체지향적인 설계** — 역할과 책임을 나누어 유지보수하기 쉬운 코드를 작성하려고 합니다. OOP, SOLID 원칙, 캡슐화, 의존성 주입(DI)을 꾸준히 학습하고 있습니다.
- 🛡️ **예외 상황 대비** — 예상하지 못한 입력이나 오류가 발생해도 서비스가 쉽게 멈추지 않도록 Exception Handling, Validation, Transaction 처리를 중요하게 생각합니다.
- 📈 **성능과 확장성 고민** — 사용자가 늘어나도 안정적으로 동작할 수 있도록 Query Optimization, Index, Caching, Scalability를 함께 고민합니다.

> 좋은 백엔드는 사용자가 특별히 의식하지 않아도 자연스럽게 빠르고 안정적인 서비스를 경험하게 만드는 것이라고 생각합니다.
